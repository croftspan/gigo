## Design Context

### Users
Claude Code power users who care about output quality and are willing to invest in setup. Developers, writers, game designers, researchers — anyone building with AI who wants better results through disciplined context engineering. They arrive via GitHub search, HN links, or the "Two Kinds of Leadership" research narrative.

### Brand Personality
**Disciplined, research-backed, direct.** No hype, no marketing-speak, no startup energy. The credibility comes from empirical evidence (9 phases of experiments) and published research (6 arXiv papers), not from claims or polish.

### Aesthetic Direction
- **Dark-mode-first.** Near-black (#0a0a0a) with subtle depth layers (#141414, #1e1e1e, #2a2a2a). Light mode supported but secondary.
- **Typography-driven.** Space Grotesk for display/headings (geometric, technical), system font stack for body (readable, familiar), mono for code. The kanji 義剛 is the brand mark.
- **Muted steel-blue accent** (#9bb8d3). Not neon, not saturated. Serious.
- **720px max-width** for containers, 680px for article body. Reading experience, not dashboard.
- **Anti-references:** Startup landing pages with gradient text, glassmorphism, hero metrics, animated backgrounds. Generic AI tool aesthetics. Marketing-speak of any kind.
- **References:** Linear's site (opinionated, professional), Stripe's docs (clean data presentation), research paper layouts (data tables, citations, pull quotes).

### Design Principles
1. **Conviction over consensus.** The site has opinions. Dark mode is the default, not a toggle afterthought. The max-width is narrow because this is a reading experience. The accent is muted because the data speaks louder than the design.
2. **Reduction as craft.** Every element earns its place. No decorative complexity. The CSS has one breakpoint (768px), two JS features (theme toggle, mobile nav), and zero framework dependencies. If it can be removed without losing value, remove it.
3. **Data before decoration.** Tables, numbers, and citations are the visual anchors. Pull quotes highlight findings, not marketing copy. The pipeline diagram is CSS boxes, not an SVG illustration.
4. **Typography hierarchy is the design system.** Three font voices (display, body, mono), eight size tokens, consistent weight usage. The hierarchy does the work that color and imagery do on other sites.
5. **Research credibility through transparency.** Every claim cites a phase or paper. Failures are reported alongside successes (Phase 2b hallucination). The eval data page shows raw numbers. Trust is built through honesty, not polish.

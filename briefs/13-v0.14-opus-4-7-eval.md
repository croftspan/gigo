## REQUIRED pre-flight fixes (do these BEFORE running the harness)

### Fix 1 — Pin the model to Opus 4.7

Current `run-eval.sh` lines 64 and 69 both run `claude -p "$prompt" --output-format json --permission-mode bypassPermissions`. No `--model` flag. This picks up whatever the CLI default is on the host, which drifts.

**Required edit:** add `--model claude-opus-4-7` to both `claude -p` invocations. Commit as a separate commit so the diff is legible:

```bash
# Bare run — temp dir with no CLAUDE.md or .claude/
(cd "$BARE_TMPDIR" && claude -p "$prompt" --model claude-opus-4-7 --output-format json --permission-mode bypassPermissions 2>/dev/null) \
  > "$DOMAIN_RESULTS/${PADDED}-bare.json" || true

# Assembled run — temp dir with CLAUDE.md and .claude/rules/
(cd "$ASSEMBLED_TMPDIR" && claude -p "$prompt" --model claude-opus-4-7 --output-format json --permission-mode bypassPermissions 2>/dev/null) \
  > "$DOMAIN_RESULTS/${PADDED}-assembled.json" || true
```

### Fix 2 — Bare-contamination glob (CRITICAL)

Memory `project_validation_suite_results.md` documents that on 2026-04-12 the bare-dir copy loop was tightened from exact-match `CLAUDE.md` to glob `CLAUDE.md*`. **That fix is not present in the current `run-eval.sh`.** It still uses exact match on line 48:

```bash
if [ "$fname" != "CLAUDE.md" ]; then
  cp -r "$f" "$BARE_TMPDIR/" 2>/dev/null || true
fi
```

Both fixtures currently contain `CLAUDE.md.original` and `CLAUDE.md.firstperson` files (verified 2026-04-19). These WILL leak into bare with the current script — the "bare" condition would have persona content.

**Required edit:** change the exclusion to a case statement that catches the glob:

```bash
for f in "$DOMAIN_DIR"/*; do
  fname=$(basename "$f")
  case "$fname" in
    CLAUDE.md*) ;;  # skip CLAUDE.md and all variants
    *) cp -r "$f" "$BARE_TMPDIR/" 2>/dev/null || true ;;
  esac
done
```

Verify after the edit: `ls $BARE_TMPDIR` right before the prompt loop and confirm no `CLAUDE.md*` files are present. (Easiest: add `echo "Bare contents:" && ls "$BARE_TMPDIR"` after the copy loop and remove before committing, OR trust the case statement and commit.)

### Fix 3 — Verify v0.14.0-beta is actually loaded

The eval runs `claude -p` in `$BARE_TMPDIR` and `$ASSEMBLED_TMPDIR`, which are bare temp dirs with no plugin context of their own. The `gigo` plugin loads from the host's `~/.claude/plugins/` regardless. Confirm:

If the version is stale, re-run the release flow from `feedback_plugin_marketplace_cache.md`.

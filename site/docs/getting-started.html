Getting Started

Requirements

Claude Code CLI installed. That's it.

Install

From marketplace

claude install @eaven/gigo

Manual

git clone https://github.com/Eaven/gigo.git
cp -r gigo/skills/ ~/.claude/skills/

First Run

Open any project and say gigo. Here's what happens:

  1. Check. GIGO looks for an existing CLAUDE.md and .claude/ directory.
  2. Kickoff. If nothing exists, a conversational brainstorm starts. You describe what you're building.
  3. Research. GIGO finds authorities in the domain and proposes blended personas — 2-3+ real practitioners per persona, each with a specific contribution.
  4. Refine. You react. "Drop that one." "Add a testing expert." "Who would they fight with?" The team takes shape through conversation, not forms.
  5. Lock in. GIGO writes CLAUDE.md, .claude/rules/, and .claude/references/.

What Gets Created

FilePurposeToken cost
CLAUDE.md Team roster and autonomy model. Auto-loaded every conversation. Always loaded
.claude/rules/ Standards, workflow, domain extensions. ~60 line cap per file, research-informed. Always loaded
.claude/references/ Deep knowledge — authority profiles, technique catalogs, checklists. Loaded on demand via "When to Go Deeper" pointers. Zero when unused

Next Steps

SkillWhen to use
gigo:plan Turn your first task into a spec and implementation plan. Brainstorms first, then structures.
gigo:execute Run the plan with agent teams or subagents. Workers get the spec, not the personas.
gigo:review Two-stage code review — spec compliance first, then engineering quality.
gigo:snap Run at session end. Audits rules for bloat and staleness, captures learnings.

Eval Data

Phase-by-phase experimental results. For the narrative version, see Two Kinds of Leadership.

Methodology

  • Domains: Rails API (OrderFlow) and Children's Novel (The Vanishing Paintings)
  • Prompts: 10 per domain across 3 axes — quality bars (A), persona voice (B), routing (C)
  • Protocol: Each prompt run twice: bare (source files only) vs assembled (full context)
  • Judging: Blinded A/B with randomized ordering, LLM-as-judge
  • Criteria: Quality bar enforcement, persona voice, expertise routing, specificity, pushback quality
  • Scale: 20 prompts × 5 criteria = 100 evaluations per run

Phase 6: Planning Pipeline (2026-03-27)

Variable: Whether brainstormer had personas loaded. Same scripted answers.

Result: Judge verdict — assembled wins decisively.

Specific assembled wins over bare:

  1. Partial unique index (prevents data integrity bug)
  2. FOR UPDATE SKIP LOCKED (better concurrency)
  3. Copy condition field (operational requirement bare missed)
  4. Pagination (bare returned unbounded results)
  5. Runnable RSpec code (bare wrote English descriptions)
  6. RESTful cancel via PATCH (bare used DELETE with query param)

Finding: Personas change how problems are explored, not just how answers are presented.
